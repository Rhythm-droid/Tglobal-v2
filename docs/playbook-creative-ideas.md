- **Hook:** Three-phase delivery loop as elastic tide: ebb (scope focus), surge (ship force), and return (measure feedback)
- **Picture it:** A horizontal, scroll-bound water-wave cross-section fills from left (ebb: low, calm) through center (surge: peak, urgent) to right (return: receding). Each phase—Scope, Ship, Measure—aligns to a wave phase. As user scrolls, a glowing cursor rides the wave, progressing through nodes. The loop repeats: wave resets (seamless), cursor re-enters. SVG paths morph smoothly; no sharp transitions. The rhythm feels tidal—inevitable, cyclic, natural.
- **Maps the loop:** Scope (ebb phase, depth 0-30% scroll): low water, flat, contemplative. Ship (surge phase, 30-70%): peak wave height, white-water churn, maximum visual density. Measure (return phase, 70-100%): receding tide, data trails, reflection. Loop: cursor releases at 100%, wave resets, next cycle begins. Repeat indefinitely.
- **Why fresh:** Tidal metaphor avoids generic orbit/spiral. Water physics (momentum, drag, surface tension) map to sprint cadence intuitively. Unlike concentric rings (overused in 2025), a sectioned wave is fundamentally directional yet cyclical—expresses forward motion within loops. SVG morph (cubic Bézier path interpolation) modern, not 3D. Scroll-scrubbed, not time-driven, so user controls tempo.
- **Build:** SVG path-based wave (cubic Bézier nodes). Morph via GSAP attr plugin: 3 path keyframes (ebb, surge, return). ScrollTrigger scrub:1 drives morphing + gradient color shift. Cursor dot on path via SVG <circle> + animationOffset. Reduced-motion: static 3-column grid showing phases side-by-side. Firefox-safe: CSS filters for glow (no blur). 60fps on mobile via GPU accelerated transforms (only <circle> moves, path static).

### Mitosis Bloom — Cell Cycle Division  `very-high` · effort M
- **Hook:** Delivery loop as biological cell division: one scope contracts, splits into ship & measure phases, recombines
- **Picture it:** A central circle (Scope) fills with scroll, then pinches at equator (cytokinesis metaphor). Two daughter circles expand outward (Ship left, Measure right), each containing mini-timelines. At 100%, circles fade and collapse back to one. Loop resets. Organic, breathing motion. Chromatin strands (thin SVG lines) animate strand-by-strand during division, adding tactile complexity without overload.
- **Maps the loop:** Scope phase (0-35% scroll): central circle grows, fills with violet. Cytokinesis (35-55%): pinch animation (scale-y: 1→0.5) at center. Ship & Measure (55-100%): two circles expand outward, anti-parallel trajectories. At loop reset, circles collapse back inward (reverse), merge, fade to single Scope circle. Next cycle identical.
- **Why fresh:** Cell division is unused in SaaS design (avoids cliché). Mitosis metaphor: one concept (scope) becomes two outcomes (ship + measure). Pinch-and-expand animation uses only CSS transform (no layout reflow), 60fps guaranteed. Chromatin strands add visual depth without bloat (10–15 lines, SVG, static paths, only opacity animates). Fresh reference: biological illustration from recent Codrops WebGL posts combines 3D camera pan with 2D SVG overlay—hybrid approach rare in 2025.
- **Build:** SVG circles (one main, two daughter via <g> clones). Pinch via CSS mask-image radial-gradient (center pinches) or GSAP scaleY transform. Chromatin SVG strands (animated opacity + stroke-dasharray). ScrollTrigger scrub:1. Reduced-motion: 3 static circle snapshots (Scope, Ship|Measure, recombined). Canvas fallback: rasterized PNG for unsupported SVG (minimal, <50KB). 60fps: transforms only, no clip-path or filter.

### Heartbeat Pulse — Systemic Rhythm  `high` · effort M
- **Hook:** Delivery loop as cardiac cycle: diastole (scope intake), systole (ship output), diastolemeasure feedback—each beat a sprint
- **Picture it:** A stylized heart chamber (SVG, organic curves) pulses 3 times per scroll section. Each pulse = one Scope→Ship→Measure sub-loop. Chamber fills (Scope: blood enters, red glow), contracts (Ship: ejection, white intensity spike), relaxes (Measure: return, cool fade). Electrical impulse traces (thin gold line) follow cardiac conduction pathways. At end of section, heart 'rests' (flat-line moment), then resumes next beat. Visual rhythm entrains user's perception.
- **Maps the loop:** Each beat cycle: 0-33% diastole (fill, chamber expands, red→violet glow, intake energy), 33-66% systole (contract, white pulse, ejection force, peak intensity), 66-100% diastolemeasure (relax, cool fade, electrical trace fades). Loop: 3 pulses across full scroll range. After scroll section ends, heart flat-lines (SVG <line> straight, no glow) for 0.5s, then resumes if user scrolls next section.
- **Why fresh:** Heartbeat is biological but rarely used in B2B design. Systemic metaphor: each delivery cycle 'keeps the business alive'. Electrical impulse (gold line following SVG path animation) adds scientific flavor without kitschy animation. Pulse animation (via SVG <animate> or GSAP) is deterministic on scroll, not emergent AI-style. References: cardiac data viz in healthcare dashboards (Codrops Feb 2026 'WebGL Medical Visualization') + biological rhythm patterns from nature docs.
- **Build:** SVG heart shape (cubic Bézier path, ~100 points, no morph—only scale/glow change). Electrical trace as SVG <polyline> (animated stroke-dashoffset) following fixed path. GSAP ScrollTrigger scrub:1 drives chamber scale (0.8→1.1→0.95) + glow color shift. Electrical line drawn via stroke-dasharray animation (0→100% over 33% scroll, repeats 3x). Reduced-motion: 3 static chamber states (diastole, systole, diastole-measure) stacked vertically. 60fps: scale + fill color only (GPU native), no clip-path.

### Murmuration Flock — Collective Emergence  `very-high` · effort L
- **Hook:** Three workflow phases as bird flock: dispersed individual scope tasks, coherent formation during ship, scatter+regroup during measure
- **Picture it:** A swarm of small circles (particles, ~30–50) starts dispersed (random positions, Scope phase). As scroll progresses, particles attract toward center (Ship phase), form a dynamic boid-like formation (teardrop or lattice), move with coordinated momentum. At Measure phase, formation explodes outward but maintains cohesion (fractal pattern). Loop: particles return to dispersed state, ready for next cycle. Trails fade behind each particle, suggesting momentum.
- **Maps the loop:** Scope (0-30%): particles dispersed, slight jitter (Brownian motion), searching. Ship (30-70%): particles converge toward center point (boid flocking rules: separation, alignment, cohesion), form emergent lattice, move collectively. Measure (70-100%): lattice expands radially, maintains connectivity (Delaunay triangulation visual), particles glow per phase (violet→accent→cool lavender). Loop reset: particles fade and respawn dispersed.
- **Why fresh:** Murmuration avoids generic orbit/spiral; it's physics-based emergence, not predetermined animation. Flocking rules (Craig Reynolds' Boids model, 1986, still trendy in 2026 creative code) feel alive because behavior emerges from simple rules. Particle trails (SVG <line> elements, opacity fade) add richness without bloat. Canvas 2D (not WebGL) keeps it lightweight; 50 particles = ~12KB. References: Codrops 'Particle Systems in Creative Code' (May 2026), Phantom Studios 'Emergence & Swarms' (FWA SOTD March 2026).
- **Build:** Canvas 2D or SVG <circle> elements (50 particles, positioned via GSAP). Boid flocking logic in `useEffect` RAF loop (separation, alignment, cohesion forces). ScrollTrigger scrub:1 drives flocking parameters: separation distance (Scope: high, Ship: low, Measure: medium), velocity scale (Scope: low, Ship: high, Measure: medium). Trails: SVG <line> elements, stroke-opacity animates. Reduced-motion: 3 static swarm states (dispersed, cohesive, expanding). 60fps: canvas-native rendering (vs DOM elements) or GPU-accelerated SVG with will-change: transform. Firefox-safe: no CSS blur on particles, only opacity.

### Seasonal Cycle — Growth & Dormancy  `high` · effort L
- **Hook:** Annual delivery rhythm as four seasons: planning (winter, dormant), growth (spring, rapid), abundance (summer, peak), harvest & rest (autumn, reflection & reset)
- **Picture it:** A stylized tree evolves through seasons as user scrolls. Winter: bare branches (thin, dark), snowfall particles (subtle, slow). Spring: buds emerge, branches thicken and color (green), rapid growth (particles ascend). Summer: full canopy (dense, golden-green glow), fruits hang (small circles on branches). Autumn: leaves fall (animated clip-path reveals), color shift warm (gold→rust), harvest baskets fill. Then reset to winter. Each season occupies 25% of scroll range. The tree's visual density + color temperature + particle direction encode the phase.
- **Maps the loop:** Scope = Winter (0-25%): bare tree, branches visible, cool palette (#1a1a2e), snowfall particles descend slowly, introspection energy. Ship = Spring+Summer (25-75%): buds bloom (opacity 0→1), canopy fills (branch thickness animates), particles ascend (y: +400px over range), golden glow intensifies. Measure = Autumn (75-100%): leaves fall (clip-path animation, random stagger), color shifts gold→rust, particles descend, harvest theme (baskets appear). Loop reset: tree enters winter dormancy, particles still, ready for next cycle.
- **Why fresh:** Seasonal metaphor aligns with natural rhythms users intuitively understand (vs. abstract geometric systems). Four-phase model (Scope-Ship-Measure + buffer/reflection phase) mirrors agile sprint retrospectives. Tree as anchor (organic, recognizable) avoids abstract shapes. Particle direction (ascend/descend/still) is data visualization: upward = growth, downward = harvest. References: David McCandless 'Information is Beautiful' (Nature + data fusion); Codrops 'Seasonal UI Transitions' (Jan 2026); natural history illustrations (Smithsonian Magazine visual style).
- **Build:** SVG tree (branches, trunk, leaf/fruit shapes). Branches animated stroke-width via GSAP. Buds/fruits via SVG <circle> opacity + scale. Leaf particles: SVG <g> elements (or Canvas 2D) with clip-path or opacity animation for 'fall'. Glow: CSS filter: drop-shadow() on tree group (Firefox-safe, <2x perf cost). ScrollTrigger scrub:1 drives all phase transitions. Reduced-motion: 4 static seasonal snapshots stacked (no animation, all visible). 60fps: stroke-width animates via GSAP (GPU-native on SVG), particles use transform: translateY (no layout thrash).
